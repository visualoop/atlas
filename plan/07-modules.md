# 07 · Modules

Every Atlas surface, in scope, before we write a line. Each module has a one-paragraph purpose, a feature list, and the keyboard surface. AI workflows live in their own doc (`08-ai-gateway.md`); Prospector and Payments are big enough to have their own files (`09-prospector.md`, `10-payments.md`).

## Today view — the home

**Purpose:** the first thing Justine sees in the morning. Not a dashboard. A queue.

**Sections (top to bottom):**

1. **Replies waiting** — count + list of conversations awaiting reply, AI-prioritized
2. **Deals rotting** — deals with no activity beyond their pipeline stage's `rotting_days` threshold
3. **Tasks due today** + overdue
4. **Calendar today** — meetings + reminders, with pre-meeting brief link
5. **AI suggested actions** — the morning brief: top 3 things to do today, generated by overnight cron
6. **Recent activity** — collapsed by default, last 24h timeline

**Keyboard:**
- `t` from anywhere → Today
- `j` / `k` to navigate sections
- `Enter` opens the focused item
- `r` jumps to Inbox-style triage

## Inbox — unified email + WhatsApp

**Purpose:** one inbox for two channels. The founder never opens Gmail or WhatsApp Business app again.

**Layout:** 3-pane (folders/labels → thread list → message)

**Folders:**
- Inbox (default, all channels)
- Sent
- Drafts
- Scheduled
- Snoozed
- Archived
- Spam (AI-filtered)
- Custom labels (per workspace)

**Channel toggle in the header:** `Email / WhatsApp / All` (default: All)

**Thread list columns:** sender, subject/preview, channel icon, timestamp, unread dot, snooze indicator, AI-suggested action chip

**Message view:**
- Thread summary at top (AI-generated for threads ≥ 5 messages)
- Per-message: sender, timestamp, body (HTML for email, plain for WhatsApp), attachments
- Right rail: contact card + company card + related deals + AI-drafted reply box

**Keyboard:**
- `i` → Inbox
- `j`/`k` → thread list nav
- `Enter` → open thread
- `r` → reply (AI-drafted if AI is on)
- `a` → archive
- `e` → snooze (modal with quick options: 1h / today / tomorrow / next week / custom)
- `p` → pin
- `c` → compose new
- `/` → focus search within inbox
- `Esc` → back to thread list

See `07-modules/email.md` (or this section continued in this file) for full spec.

## Contacts + companies

**Purpose:** the graph. Every person, every company, with their full history.

**List view:** dense table, filterable by lifecycle stage / owner / tag / source. Search-as-you-type.

**Slide-over detail (right panel):**
- Header: name, company, contact info, last activity timestamp
- Quick actions: email, WhatsApp, call (click-to-call future), new task, new note, new deal
- Tabs:
  - **Timeline** (default) — chronological, AI summary at top
  - **Conversations** — email + WhatsApp threads
  - **Deals** — open + closed
  - **Notes** — pinned + chronological
  - **Files** — uploaded + extracted
  - **Tasks** — open + completed
  - **AI memory** — facts Atlas has learned ("prefers WhatsApp", "decision-maker", etc.)
- Footer: tags, custom fields, owner, created date

**Keyboard:**
- `c` → Contacts list
- `Shift+C` → New contact (slide-over form)
- `Enter` → open detail
- `e` → edit
- `n` → new note on this record
- `t` → new task on this record
- `Esc` → close slide-over

## Pipelines + deals

**Purpose:** sales motion that adapts per workspace.

**Pre-seeded shapes:**

- **Omnix License Funnel:** Cold → Trial Requested → Demo Booked → Demo Held → Pricing Sent → Negotiation → License Sold → Onboarding → Active → Renewal Due → Renewed / Churned
- **Studio Project Funnel:** Inbound → Discovery Call → Scope → Proposal Sent → Negotiation → Deposit Paid → Building → Handover → Closed-won / Closed-lost
- **Marketplace Creator Funnel:** Signed Up → First Listing Drafted → First Listing Published → First Sale → 5 Sales → 10 Sales → Established → Churned / Dormant

**Views:**
- **Kanban** (default) — drag/drop with optimistic + Undo, per-stage WIP limits optional
- **Table** — TanStack Table, filter/sort/group, inline edit
- **Timeline** — Gantt-like view of expected close dates (future)

**Deal record:** title, amount, currency (KES default), pipeline + stage, owner, contact, company, source, expected close, custom fields, AI fit score, win/loss reason

**Stage automations** (configurable per stage):
- On enter stage X: send template Y, create task Z, post to a Slack-equivalent (future), notify someone
- On idle > N days in stage: flag as rotting, AI suggests next action

**Keyboard:**
- `p` → Pipelines (default workspace pipeline)
- `Shift+D` → New deal
- `1`–`9` → quick filter by stage
- `Enter` → open deal slide-over
- `←` `→` → move deal to previous/next stage (when focused)

## Documents — proposals, quotes, invoices, contracts, briefs

**Purpose:** every client-facing artifact, generated by Atlas, signed in Atlas, paid in Atlas.

**Kinds:**

- **Proposal** — scoped pitch (Studio)
- **Quote** — line-item quote (Omnix, Studio)
- **Invoice** — billable, links to Payment Request (Studio, Omnix)
- **Contract** — legal binding, e-signature required (Studio engagement letter, Omnix license agreement)
- **Brief** — internal scoping doc

**Template library** per workspace:
- Studio: "Engagement letter", "Scope of work", "Final invoice"
- Omnix: "License agreement", "Sales quote", "Renewal invoice"
- Shared: blank document

**Editor:** TipTap rich text with **locked sections** (legal copy, signature blocks). Inline variable insertion (`{{deal.amount}}`, `{{client.name}}`, `{{date}}`).

**AI:**
- "Generate proposal for this deal" → uses deal + company + last 5 conversations → drafts the body
- "Critique this proposal" → flags scope creep, pricing inconsistencies, missing terms
- "Convert quote to invoice" → preserves line items, adds payment terms

**PDF generation:** `@react-pdf/renderer` server-side. Identical styling to the on-screen document. Embed Paystack pay link as a button on page 1 + QR code on footer for invoices.

**Public share:** `/share/{token}` — unauth view with sign / pay actions. View tracked, signed tracked, paid tracked.

**E-signature:** DocuSeal embedded via iframe at `/share/{token}/sign`. Audit trail returned via webhook.

**Send via:**
- Email (Atlas inbox compose, document auto-attached as PDF + link to web view)
- WhatsApp (document text-summary + PDF attached + link to web view)
- Just copy public link

**Keyboard:**
- `d` → Documents list
- `Shift+D` (when not in pipeline) → New document (with kind picker)
- `s` → Send (opens compose with document attached)

## Campaigns + sequences

**Purpose:** outbound at scale without the spam smell.

**Campaign types:**

- **One-shot broadcast** — single message to an audience
- **Sequence** — multi-step drip (e.g., 3 emails over 10 days)
- **Trigger-based** — fires when a contact enters a state (e.g., "joined the cold stage")

**Channels:** email + WhatsApp (separate campaigns, or multi-channel sequences mixing both)

**Audience builder:** filter by lifecycle stage, source, tag, custom field, last activity, fit score

**Personalization:** `{{contact.firstName}}` + AI-personalized first line ("Hi Patricia, I noticed your studio specialises in office partitioning…")

**Suppression:**
- Per-workspace + global opt-out list
- Auto-suppress: already-customer, do-not-contact, bounced
- Reply detected → sequence pauses, escalates to inbox

**A/B testing** — subject lines, send times, first-line variants. Statistical significance gate before declaring a winner.

**AI pre-send critic** — flags spam-trigger phrases, scope creep, broken merge tags

**Send-time optimizer** — AI picks best time per contact based on past open behavior

**Keyboard:** `Shift+B` → New broadcast/sequence

## Analytics

**Purpose:** decisions, not decoration.

**Pre-built dashboards per workspace:**

- **Omnix:**
  - License funnel conversion (cold → trial → demo → sold)
  - Module attach rate (% buying Dawa vs Soko Retail vs both)
  - Renewal pipeline (next 30 / 60 / 90 days)
  - Support ticket trend (when we have support module)
  - LTV by module

- **Marketplace:**
  - GMV (today / this week / month / YTD)
  - Take-rate trend
  - Top creators by GMV
  - New creator activations (signed up → first sale)
  - Churn signals (creators with declining GMV)
  - Subscription MRR

- **Studio:**
  - Pipeline value (open) by stage
  - Project P&L (when project costing wired in)
  - Win rate over time
  - Average deal cycle (days from inbound to close)
  - AR aging (invoiced - paid, by bucket)
  - Capacity vs commitments

**AI-generated reports:**
- Daily digest (email + WhatsApp + in-app, 7am Africa/Nairobi)
- Weekly business review (Friday 4pm) — proper narrative doc
- Monthly forecast (1st of month) — revenue projection with confidence interval

**Custom reports:** "Ask Atlas" — type a question, AI runs Drizzle queries, returns table + chart. (Built on top of the AI tool registry.)

**Export:** CSV + PDF download. Scheduled email of any report.

**Keyboard:** `a` → Analytics

## Calendar + meetings

**Purpose:** stop using Google Calendar separately.

**Features:**
- Two-way sync with Google Calendar (per-user OAuth)
- Microsoft Calendar (future)
- Meeting booker — single + group, with available slots from calendar
- iCal feed per workspace (subscribe URL for shared calendars)
- Time-zone aware (Africa/Nairobi default)
- Pre-meeting brief (AI-generated 1 hour before)
- Post-meeting summary + action items extraction (paste transcript or upload audio → Whisper)
- Recurring meetings

**Booking page** per workspace:
- Public URL: `book.blyss.co.ke/<workspace>/<meeting-type>`
- Branded with workspace settings
- Slots computed from Justine's connected calendar
- Submission creates a meeting + sends ICS + confirmation email/WhatsApp

**Keyboard:** `k` → Calendar (because `c` is Contacts, `m` is reserved)

- Bulk complete, bulk reschedule

## Notes

Sub-feature; standalone search at `Shift+N`. TipTap editor. Vector indexed for search. Linked to records by `related_to_type`/`related_to_id`.

## Files

Sub-feature; standalone list at `Shift+F`. R2 storage. Extracted text for PDF/image (Gemini Flash extraction). Searchable. Drag/drop to upload.

## Search — universal

`⌘K` opens the palette. Type a query, see hybrid results across all resources (contacts, companies, deals, conversations, messages, notes, files, documents, tasks, timeline events). Group by type. Quick-action keyboard shortcuts beside each result.

## Onboarding + first-run

**First-time Org Owner:**

1. Sign up → email verified → land on `Welcome to Atlas`
2. Wizard:
   - Org name (default: from Google name if OAuth)
   - First workspace name (or pick from 3 templates: "Sales motion", "Project work", "Marketplace")
   - Brand color (optional, defaults to Atlas accent)
   - Skip or "Set up email" (Resend wizard)
3. Lands on Today view with empty states + "Get started" checklist
4. Checklist (dismissible):
   - [ ] Connect Resend → can send email
   - [ ] Add a Paystack key → can take payments
   - [ ] Run your first Prospector search
   - [ ] Add an AI provider key → unlock AI features
   - [ ] Import contacts (CSV or Gmail)
   - [ ] Invite a team member

**First-time invited member:**

See invitation flow in `06-auth-and-permissions.md`.

## Mobile / responsive

Atlas is keyboard-first, but the mobile view exists for on-the-go:

- Inbox triage (read + reply via WhatsApp template / canned email)
- Today view
- Contact lookup
- Deal view (read-only or minor edits)
- Document signing approval

Mobile uses Vaul drawers instead of slide-overs. Touch-friendly targets (44px min). Most heavy UI (Kanban drag, table dense view, document editor) collapses to a "View on desktop" prompt.

## Public surfaces (unauth)

- `/share/{token}` — view a document
- `/share/{token}/sign` — sign a document (DocuSeal embed)
- `/share/{token}/pay` — pay an invoice (Paystack hosted)
- `/book/{workspace}/{meeting-type}` — book a meeting (public scheduler)
- `/invite/{token}` — accept an invitation
- `/login`, `/sign-up`, `/verify-email`, `/forgot-password`

That's the entire public surface. No marketing pages, no docs site (yet — docs come post-launch at `docs.atlas.blyss.co.ke`).
